# 🚀 Profile README — Deployment & Customization Guide

This package turns your GitHub profile into a premium, animated, mission-control
dashboard. Everything is **production-ready**. Follow the steps below.


## 🧭 The Key Concept: the special profile repo

Your README only shows on your profile if it lives in a repository **named exactly
the same as your username**:

```
Repository name:  Andy-XO/Andy-XO
```

If that repo doesn't exist yet, create it (it must be **Public**, and GitHub will
offer a "you found a secret" hint when the name matches your username).

## ⚡ Quick Deploy (3 commands)

From inside the `github-profile/` folder:

```bash
# 1. Initialize and point at YOUR profile repo
git init
git add .
git commit -m "feat: premium animated profile README"
git branch -M main

# 2. Connect to the special repo (create Andy-XO/Andy-XO on github.com first)
git remote add origin https://github.com/Andy-XO/Andy-XO.git

# 3. Push
git push -u origin main
```

Open `https://github.com/Andy-XO` — the profile is live. ✨

> **Note:** the contribution snake and (optional) metrics images will show as
> broken for a few minutes until their Actions run for the first time. That's
> expected — see below.

## 🐍 Enable the Snake Animation

The snake SVG is generated by a GitHub Action — it isn't a static file.

1. After pushing, go to the repo's **Actions** tab → enable workflows if prompted.
2. Open **"Generate Snake Animation"** → **Run workflow** (manual first run).
3. It creates an `output` branch containing `github-snake-dark.svg`.
4. The README already points at:
   `https://raw.githubusercontent.com/Andy-XO/Andy-XO/output/github-snake-dark.svg`

After that it auto-refreshes every 12 hours. No token required.

## 📊 (Optional) Enable GitHub Metrics

The metrics dashboard needs a token because it reads richer data.

1. Create a token → https://github.com/settings/tokens (classic)
   - Scopes: **`repo`**, **`read:user`**
2. Repo → **Settings → Secrets and variables → Actions → New repository secret**
   - Name: `METRICS_TOKEN`
   - Value: *(your token)*
3. Run the **"GitHub Metrics"** workflow once.
4. Add this line to README.md wherever you want it (e.g. in the Analytics section):
   ```markdown
   <p align="center"><img width="95%" src="./assets/metrics.svg" alt="GitHub Metrics"/></p>
   ```

Skip this entirely if you don't want it — the rest of the profile is unaffected.

## 🎨 Customization Guide

### Change the accent color
The theme is built on three cyan/blue stops. To re-skin globally, find/replace
these hex codes across `README.md`, `assets/*.svg`, and `.github/workflows/snake.yml`:

| Role | Hex |
|------|-----|
| Primary (cyan) | `00D9FF` |
| Secondary (blue) | `33A1FF` |
| Tertiary (indigo) | `6E7BFF` |
| Background | `05070D` |
| Muted text | `9FB4C7` |

Example: for a violet theme, swap `00D9FF → A78BFA`, `33A1FF → 8B5CF6`, `6E7BFF → 6366F1`.

### Edit the typing animation
In README.md, the `readme-typing-svg` URL has a `lines=` query param — change the
text segments (use `;` to separate lines, `%C2%B7` is the `·` separator).

### Edit the hero / footer banners
`assets/hero-banner.svg` and `assets/footer-wave.svg` are plain, commented SVG.
Change the `<text>` content, the gradient `<stop>` colors, or animation `dur=`
values directly. They animate via SMIL, which GitHub renders natively.

### Reorder / remove sections
Each section is fenced with a banner comment like:
```
<!-- SECTION · FEATURED PROJECTS -->
```
Cut/paste whole blocks to reorder. The `01 ⟶`, `02 ⟶` numbering is just text —
renumber as you like.

### Add a real project link
In the Featured Projects table, wrap a card title in a link:
```markdown
### [🛰 VoltAeroTech](https://github.com/Andy-XO/voltaerotech)
```
